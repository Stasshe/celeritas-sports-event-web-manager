# Admin UI intent

管理画面、長時間編集する作業場。装飾より情報密度と位置の安定を優先する。

サイドバー、header、本文の寸法依存を分離しない。同じlayout treeで領域を確保し、重なりをoffset計算で避けない。狭幅だけnavigation表現を変え、機能階層は変えない。

page固有の箱と余白を増やさない。共通layoutが外周を決め、pageは内容の区切りだけ持つ。

トーナメント表はPCで全体関係を静的表示する。表示面積が限られるスマートフォンだけpageから独立したviewportとし、背景操作で注視範囲を移動・拡大する。試合カード操作は結果入力へ専有する。

編集は入力を止めずlocal stateへ反映する。保存feedbackのために作業画面全体を更新しない。自動保存は画面や入力種別ごとに挙動を変えず、モーダルを含む連続編集を一つのwriteへまとめる。保存中の追加入力を古い保存完了で確定扱いにしない。
